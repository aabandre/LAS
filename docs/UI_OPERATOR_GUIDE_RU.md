# Local Admin Scanner — инструкция оператора (RU)

## 1) Главная страница (`/`)

### 1.1 Блок «Настройки подключения»
- **Контроллер домена / GC/DC серверы / Domain / NetBIOS** — параметры подключения к AD.
- **Пользователь / Пароль** — основная учётная запись сканирования.
- **Логин/пароль для серверов** — отдельная УЗ для server-target (если нужно).
- **Макс. потоков, таймауты, лимиты network/RPC** — управление скоростью и стабильностью.

### 1.2 OU-конструкторы
- **Конструктор OU рабочих станций / серверов** помогает собрать DN.
- Кнопка **«Собрать DN»** формирует DN из короткого пути.

### 1.3 Фильтры и охват
- **Разрешённые админы** — по одной записи на строку (`user` или `DOMAIN\user`).
- **Доп. алиасы доменов** — по одной записи на строку (учитываются в risk-мэтчинге).
- **Включить/Исключить объекты** — маски `*` и `?`.
- **Фильтр по ОС** — содержит строку.

### 1.4 Локальные группы и runtime-параметры
- Выберите группы: `Administrators`, `RDP Users`, `DCOM Users`, `Remote Management Users`.
- **Разворачивать доменные группы через LDAP** — _по умолчанию выключено_ (замедляет скан).
- **RPC fallback / WMI fallback / debug-log** — для сложных/legacy сетей.

### 1.5 Кнопки
- **Запустить** — старт сканирования.
- **Остановить** — мягкая остановка текущего запуска.
- **Сохранить / Загрузить** — сохранение и загрузка настроек (localStorage браузера).
- **Stable Fast / Надёжный быстрый** — готовые пресеты.

## 2) Прогресс

- KPI: просканировано, всего, успех, ошибки, участники, хост/мин, ETA.
- Прогресс-бар + текущий хост.
- Кнопки артефактов: **JSON / CSV / Summary / Сводка**.

## 3) Сводка и аналитика (`/summary`)

### 3.1 Верхняя панель
- **Печать**
- **Экспорт отфильтр. админов (CSV)**
- **Загрузить предыдущую сводку**
- **Пересчитать**
- **Скан 1 / Скан 2 / Сравнить**

### 3.2 Вкладка «Админы»
- Поиск, фильтры, сортировка, выбор плотности (Компакт/Удобный), пагинация.
- Массовое выделение:
  - чекбокс в заголовке = выбрать видимые строки.
  - чекбоксы в строках = точечный выбор.
- **В очередь выбранное** — добавляет выбранные записи в remediation queue.
- **В очередь** в строке — добавляет конкретную запись.

### 3.3 Очередь исправления (remediation)
- Поля **DOMAIN / User / Password** (опционально).
  - если заполнены, `Run now` выполняется под этими учётными данными;
  - если пусто — используется контекст по умолчанию.
- Кнопки:
  - **Выполнить** — удаление по API для всех записей в очереди.
  - **Экспорт .ps1** — выгрузка скрипта исправления.
  - **Очистить** — очистка очереди.

### 3.4 Вкладки «Риск / Чистые / Карта / Сравнение / История»
- **Риск**: быстрые фильтры уровня риска.
- **Карта**: тепловая карта учёток по машинам.
- **Сравнение**: дельта между двумя сводками.
- **История**: просмотр/загрузка файлов результатов.

---

## 4) Практика эксплуатации

1. Запускайте с **минимально необходимой** сервисной УЗ.
2. Для `Run now` используйте отдельную админскую УЗ только при необходимости.
3. На больших доменах:
   - держите LDAP-экспанд выключенным;
   - включайте точечно на ограниченном scope.
4. Всегда проверяйте очередь перед выполнением удаления.
